# 用户故事

## 用户角色
- 经常填写人员申报内容的个人用户。
- 代他人整理并填写资料的工作人员。
- 需要先汇总多人信息，再统一填报的组织者。

## 用户故事列表

### 1. 填写自己的申报信息
作为一个经常要提交人员资料类表单的用户，
我希望系统能根据要求自动整理并填写我的信息，
以便减少重复输入，提高填表效率。

### 2. 填写别人的申报信息
作为一个需要代他人填写人员资料类表单的用户，
我希望系统能从已有资料中提取对方信息，
以便更快完成代填工作。

### 3. 用 Excel 汇总多人信息
作为一个需要统一收集资料的组织者，
我希望能够从已有资料中汇总别人的信息，
以便集中管理并支持后续填报。

### 4. 从微信聊天记录中提取信息
作为一个资料分散在微信中的用户，
我希望系统可以利用导出的聊天记录和下载文件夹中的附件提取信息，
以便把零散内容纳入统一资料库。

## 关键需求
- 能从多个来源定位个人资料。
- 能按人汇总资料，而不是按文件汇总。
- 能显示每条信息对应的来源。
- 能标出冲突信息，方便人工核对。
- 能在已有资料未变化时跳过重复扫描，减少等待时间。

## 验收关注点
- 用户能快速找到某个人的完整资料。
- 用户能判断每条信息来自哪里。
- 用户能发现并处理冲突字段。
- 用户不需要修改原始资料即可完成信息整理。
