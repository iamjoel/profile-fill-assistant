# 页面清单

本文档整理 MVP 阶段的页面/工作区，供产品、设计、前端实现统一对齐。

## 页面总览

MVP 采用 4 个一级页面 / 工作区：

1. 扫描入口页
2. 人员列表页
3. 人员工作台
4. 待填文件辅助页

说明：
- “右侧问答栏”不是独立页面，而是“待填文件辅助页”中的固定侧栏区域。
- “人员工作台”内部包含 3 个视图切换：资料概览、冲突核对、来源文件。

## 1. 扫描入口页

页面目标：
- 完成扫描源配置。
- 发起全量扫描或增量扫描。
- 让用户理解最近一次扫描做了什么。

进入方式：
- 首次启动默认进入。
- 从全局导航进入“扫描”。

页面内容：
- 扫描源选择区：展示当前已选择的本地目录。
- 扫描模式区：支持首次全量扫描、增量扫描。
- 扫描结果摘要区：展示最近一次扫描时间、处理文件数、跳过文件数、失败文件数、聚合后人数。
- 最近任务区：展示最近一次任务状态和异常提示。

关键操作：
- 选择或更换扫描目录。
- 发起全量扫描。
- 发起增量扫描。
- 查看失败文件和失败原因。

关键状态：
- 首次未配置时，引导用户先选择目录。
- 扫描进行中时，展示明确进度状态。
- 扫描完成后，给出进入“人员列表页”的下一步入口。
- `workspace/` 初始化失败时，明确报错。

## 2. 人员列表页

页面目标：
- 让用户以“人”为单位快速找到目标对象。

进入方式：
- 扫描完成后推荐进入。
- 从全局导航进入“人员”。

页面内容：
- 列表工具区：关键词搜索、状态筛选、排序。
- 人员列表区：展示姓名、资料完整度、冲突数、最近更新时间、主要身份线索摘要。
- 快速预览区或行内摘要：展示基础信息摘要和高风险提醒。

关键操作：
- 按姓名、手机号、证件号、单位等线索搜索。
- 按“有冲突”“待补充”“最近更新”筛选。
- 点击某个人进入人员工作台。

关键状态：
- 无扫描结果时，提示先去扫描入口页建立资料库。
- 搜索无结果时，提示调整关键词。
- 冲突人数较多时，默认支持按冲突优先排序。

## 3. 人员工作台

页面目标：
- 围绕单个人完成查看、核对、确认和追溯。

进入方式：
- 从人员列表页点击进入。

页面结构：
- 顶部基础信息区：展示姓名、别名、主要身份线索、更新时间、当前资料状态。
- 主内容区包含 3 个视图切换：
  - 资料概览
  - 冲突核对
  - 来源文件

### 3.1 资料概览

页面目标：
- 快速查看该人的结构化资料现状，判断哪些字段可直接用于填报。

页面内容：
- 基础信息、教育信息、职业信息、其他信息四个分组。
- 每个字段展示字段值、字段状态、是否人工确认、最近确认时间。
- 对 `missing`、`candidate`、`conflict` 字段做显式区分。

关键操作：
- 点击字段查看来源。
- 对可确认字段执行“确认使用”。
- 对缺失字段标记“待补充”。

关键状态：
- `confirmed` 字段应作为默认最稳定展示状态。
- `conflict` 字段不能伪装成已确认值。

### 3.2 冲突核对

页面目标：
- 让用户集中处理冲突字段。

页面内容：
- 冲突字段列表，按字段分组展示多个候选值。
- 每个候选值展示来源、抽取时间、原文片段或定位信息。
- 当前已选最终值与未决状态清晰区分。

关键操作：
- 从候选值中选择最终值。
- 保持未决，稍后处理。
- 跳转查看对应来源文件。

关键状态：
- 所有冲突字段都必须可见。
- 用户完成选择后，资料概览中的字段状态要立即同步。

### 3.3 来源文件

页面目标：
- 让用户确认某条信息的来源，建立对系统结果的信任。

页面内容：
- 来源文件列表：展示文件名、路径、类型、抽取状态、最近抽取时间。
- 文件详情：展示该文件贡献过哪些字段。
- 失败文件也应在该视图中可见。

关键操作：
- 按文件类型或抽取状态筛选来源。
- 从字段反查来源文件。
- 定位到转换后的 `docx` 文件。

关键状态：
- 失败文件必须展示失败状态。
- 若字段来自多个文件，应完整展示全部来源。

## 4. 待填文件辅助页

页面目标：
- 在用户实际填写文件时，基于当前文件内容提供相关人员资料、来源和问答辅助。

进入方式：
- 用户主动打开某个待填文件时进入。

页面布局：
- 左侧为待填文件预览区。
- 中间为候选人员与资料摘要区。
- 右侧为问答栏。

### 4.1 待填文件预览区

页面内容：
- 当前打开文件内容的可解析预览。
- 从文件中提取出的候选人线索、单位、学校、手机号、证件号等上下文。
- 当前文件类型与解析状态提示。

关键操作：
- 查看当前文件中已识别出的线索。
- 在多个候选人之间切换查看匹配结果。

关键状态：
- 文件可打开但解析不完整时，明确提示“部分可解析”。
- 无法可靠识别目标人时，显示“不确定”，并给出候选人而不是默认绑定。

### 4.2 候选人员与资料摘要区

页面内容：
- 当前最相关候选人的基础摘要。
- 关键字段状态，尤其是可直接填报项、冲突项、缺失项。
- 与当前文件最相关的来源提示。

关键操作：
- 在候选人之间切换。
- 一键跳转到对应人员工作台。

关键状态：
- 默认排序优先展示与当前文件上下文匹配度最高的人。
- 若无任何可匹配人员，明确提示“未找到可关联资料”。

### 4.3 右侧问答栏

页面目标：
- 让用户围绕当前待填文件持续提问，并获得带来源的答案。

页面内容：
- 默认推荐问题或资料摘要。
- 用户输入框。
- 多轮回答记录。
- 每条回答对应的来源和冲突提示。

关键操作：
- 提问当前字段相关问题。
- 追问来源、冲突原因、缺失项。
- 从回答中跳转到来源文件或人员工作台。

关键状态：
- 回答中必须区分“来源事实”和“模型推断”。
- 若答案依赖冲突字段，优先展示冲突，而不是给出伪确定答案。
- 若当前上下文不足以回答，明确说明缺少什么信息。
