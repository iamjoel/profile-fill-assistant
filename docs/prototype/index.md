# 页面说明

本文档根据 `docs/prototype` 下的三张原型图整理页面描述，供产品、设计、前端实现统一对齐。

对应原型：
- `dashboard.png`：仪表盘与扫描文件夹流程。
- `people-manage.png`：人员信息管理与人员详情抽屉。
- `fill-the-form.png`：从仪表盘进入填表辅助流程。

## 页面总览

当前原型采用左侧导航 + 顶部品牌栏的桌面应用布局。

一级导航包含 3 个页面：

1. 仪表盘
2. 已填表格
3. 人员信息

全局区域：
- 顶部品牌区展示应用图标、产品名“表姐不加班”和一句副标题。
- 右上角提供扫描入口图标和设置图标。
- 左侧导航固定展示页面入口。
- 左下角展示当前管理员账号信息。

关键流程：
- 扫描文件夹：从右上角扫描图标触发。
- 填表辅助：从仪表盘右上角“填表”按钮触发。
- 人员管理：从“人员信息”页面查看、编辑和新增人员资料。

## 1. 仪表盘

页面目标：
- 汇总当前资料库规模。
- 展示已扫描文件夹。
- 提供“扫描文件夹”和“填表”的主要入口。

进入方式：
- 应用启动后默认进入。
- 从左侧导航点击“仪表盘”进入。

页面内容：
- 数据概览卡片：
  - 已填表格数量。
  - 人员数量。
  - 已扫描文件夹数量。
- 已扫描文件夹列表：
  - 展示本地文件夹路径。
  - 展示最近一次扫描完成时间。
  - 提供“重新扫描”操作。
- 顶部操作：
  - 右上角扫描图标。
  - 设置图标。
  - “填表”按钮。

关键操作：
- 点击右上角扫描图标，打开扫描文件夹弹窗。
- 点击“填表”，打开选择待填表格弹窗。
- 点击“重新扫描”，重新扫描对应文件夹。

关键状态：
- 无扫描文件夹时，应在已扫描文件夹区域提示用户先添加扫描目录。
- 已扫描文件夹存在时，展示路径、扫描完成时间和重新扫描入口。
- 数据卡片应随扫描和填表结果更新。

## 2. 扫描文件夹流程

触发入口：
- 在仪表盘右上角点击扫描图标。

### 2.1 选择扫描文件夹

页面形态：
- 居中弹窗。
- 背景页面加遮罩。

弹窗内容：
- 标题“扫描文件夹”。
- 文件夹路径输入框。
- “浏览...”按钮。
- 文件类型说明：只扫描文件夹中的 Doc、PDF、Excel、MD 和 TXT 文件。
- “取消”和“扫描”按钮。

关键操作：
- 点击“浏览...”选择本地文件夹。
- 点击“扫描”开始扫描。
- 点击关闭图标或“取消”关闭弹窗。

关键状态：
- 未选择文件夹时，“扫描”按钮应避免进入无效扫描。
- 选择文件夹后，路径输入框展示完整路径。
- 用户取消后返回仪表盘，不改变已有扫描结果。

### 2.2 扫描进行中

页面形态：
- 居中进度弹窗。
- 背景仍停留在仪表盘。

弹窗内容：
- 标题“正在扫描”。
- 当前扫描文件夹路径。
- 扫描进度条和百分比。
- 当前正在处理的文件名。
- 已扫描文件数 / 总文件数。
- 文件处理列表，按文件展示类型图标、文件名和状态。

文件状态：
- 已完成。
- 扫描中。
- 等待中。

关键状态：
- 扫描过程中应展示明确进度，例如 65%。
- 当前文件应单独展示，便于用户知道扫描没有卡死。
- 列表中至少展示最近处理或当前队列中的文件。
- 扫描完成后，仪表盘的统计卡片和已扫描文件夹列表应更新。

## 3. 人员信息页

页面目标：
- 以人员卡片方式管理资料库中的人员。
- 支持搜索、新增、预览和编辑人员信息。

进入方式：
- 从左侧导航点击“人员信息”进入。

页面内容：
- 页面标题“人员信息”。
- 人员总数。
- 搜索框，支持按姓名、电话、邮箱或证件号搜索。
- “新增人员”按钮。
- 人员卡片网格。
- 右侧人员详情抽屉。

人员卡片内容：
- 头像。
- 姓名。
- 职位或身份。
- 手机号，按隐私规则脱敏展示。
- 邮箱。
- 证件号，按隐私规则脱敏展示。
- 资料状态，例如“已完善”。
- 最近更新时间。
- “预览”和“编辑”操作。

关键操作：
- 输入关键词搜索人员。
- 点击“新增人员”创建新人员资料。
- 点击“预览”打开右侧详情抽屉。
- 点击“编辑”进入人员资料编辑状态。

关键状态：
- 无人员时，提示可通过扫描或新增人员建立资料库。
- 搜索无结果时，提示调整关键词。
- 人员资料完整时展示“已完善”状态。
- 隐私字段在列表卡片中默认脱敏。

## 4. 人员详情抽屉

触发入口：
- 在人员信息页点击人员卡片上的“预览”。

页面形态：
- 右侧滑出抽屉。
- 抽屉顶部展示“人员信息 · 姓名”。
- 右上角提供关闭按钮。

抽屉内容：
- 顶部标签：
  - 预览。
  - 编辑。
- “下载 MD”按钮。
- Markdown 风格资料预览。

资料分组：
- 基础信息：姓名、性别、手机号、邮箱、证件号、出生日期、地址。
- 教育信息：最高学历、学位、毕业院校、专业、毕业时间。
- 职业信息：单位、部门、岗位、职称、职称取得时间。
- 其他信息：证书、荣誉、备注。

关键操作：
- 在“预览”和“编辑”之间切换。
- 下载当前人员资料的 Markdown 文件。
- 关闭抽屉返回人员列表。

关键状态：
- 预览态以只读形式展示完整资料。
- 编辑态应允许修改对应字段。
- 下载 MD 时，应导出当前人员详情中的结构化内容。

## 5. 填表辅助流程

触发入口：
- 在仪表盘点击“填表”按钮。

### 5.1 选择待填表格

页面形态：
- 居中弹窗。
- 背景页面加遮罩。

弹窗内容：
- 标题“选择要填的表格”。
- 文件拖拽上传区域。
- 支持格式说明：Doc、PDF、Excel、MD、TXT 文件。
- “取消”和“确定”按钮。

关键操作：
- 将文件拖拽到上传区域。
- 点击上传区域选择文件。
- 点击“确定”进入填表辅助工作区。
- 点击关闭图标或“取消”返回仪表盘。

关键状态：
- 未选择文件时，“确定”按钮应避免进入下一步。
- 选择不支持的文件类型时，应明确提示。
- 成功选择文件后，应保留文件名并进入预览准备状态。

### 5.2 填表辅助工作区

页面目标：
- 左侧预览待填文件。
- 右侧通过 AI 助手获取和补全人员资料。

页面布局：
- 顶部显示当前文件名，例如“个人信息表.docx”。
- 左侧为文件预览区。
- 右侧为 AI 助手聊天区。

文件预览区内容：
- 文档页码控制，例如 `1 / 3`。
- 缩放控制，例如 `100%`、放大、缩小。
- 下载、打印、全屏等文档工具。
- 待填表格内容预览。
- 表格字段和空白填写线。

AI 助手区内容：
- 标题“AI 助手”。
- “清空对话”按钮。
- 对话消息列表。
- 底部输入框和发送按钮。

关键操作：
- 在右侧输入人员姓名或待补全字段问题。
- AI 根据当前文件和资料库返回可填信息。
- 用户继续追问或清空对话。
- 在左侧查看表格字段，结合右侧回答完成填写。

关键状态：
- 刚进入时，AI 主动询问“请问要填充谁的信息？”。
- 用户输入姓名后，AI 显示处理中状态。
- 当前文件需要可滚动预览。
- 文档工具栏应保持可用，便于翻页、缩放、下载、打印和全屏查看。

## 6. 已填表格页

原型导航中包含“已填表格”页面，但三张图未展示该页面细节。

基于导航语义，该页面应承载：
- 已完成或已生成的表格记录列表。
- 文件名、关联人员、生成时间和状态。
- 打开、下载、重新填充或删除记录等操作。

该页面的具体布局和交互需以后续原型为准。
